# Requirements to User Stories - Prompt Templates

## Template 1: Basic Conversion

```text
Convert the following software requirement into user stories using the format:
As a [user], I want [goal], so that [benefit].

Requirement:
[Insert requirement here]
```

## Template 2: Conversion with Acceptance Criteria

```text
Act as a business analyst. Convert the following requirement into user stories and acceptance criteria.

Requirement:
[Insert requirement here]

Return the output as:
1. User Story
2. Acceptance Criteria in Given-When-Then format
3. Assumptions
4. Open Questions
```

## Template 3: AI Native SDLC Research Prompt

```text
Analyze the following requirement from the perspective of AI Native SDLC.
Identify:
- Candidate user stories
- Acceptance criteria
- Ambiguities
- AI assistance opportunities
- Possible automation points

Requirement:
[Insert requirement here]
```

## Example Requirement

```text
The application should allow registered users to upload documents and search within them using natural language queries.
```

## Expected SDLC Value

These prompts support requirements refinement and help convert informal descriptions into structured engineering artifacts.
