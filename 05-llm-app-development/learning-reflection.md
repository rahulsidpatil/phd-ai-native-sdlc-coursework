# Learning Reflection - LLM App Development

## Summary

This module helped me understand how LLM capabilities can be packaged into usable applications for software engineering workflows.

## Key Learnings

1. LLM app development requires more than prompt writing.
2. Architecture should separate prompt building, model access, context retrieval, validation, and logging.
3. AI-generated SDLC artifacts should be reviewed before use.
4. Risks such as hallucination, data leakage, and poor traceability must be controlled.

## Connection to PhD Research

LLM app development is central to AI Native SDLC because it shows how AI capabilities can be embedded into development lifecycle activities such as requirements, testing, coding, documentation, and maintenance.
