# LLM App Development Use Cases for AI Native SDLC

## 1. Requirements Assistant

Converts informal requirements into structured user stories, acceptance criteria, assumptions, and open questions.

## 2. Test Design Assistant

Generates initial functional, negative, and boundary test cases from user stories or acceptance criteria.

## 3. Code Explanation Assistant

Explains code snippets, summarizes logic, and creates documentation drafts.

## 4. Documentation Assistant

Generates release notes, API documentation drafts, user guides, and onboarding material.

## 5. Defect Analysis Assistant

Analyzes bug reports and suggests reproduction steps, possible causes, and impacted modules.

## 6. Knowledge Assistant

Uses RAG to answer questions from project documents, architecture records, test plans, and engineering standards.

## Research Relevance

These use cases demonstrate how AI can become a native part of SDLC workflows rather than an external add-on.
