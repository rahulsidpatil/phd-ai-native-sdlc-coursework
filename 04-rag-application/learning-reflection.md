# Learning Reflection - RAG Application

## Summary

This module helped me understand how retrieval can improve the usefulness of LLM applications by grounding responses in external knowledge.

## Key Learnings

1. RAG is useful when answers must be based on project-specific documents.
2. Retrieval quality strongly affects answer quality.
3. Source traceability is important for academic and software engineering use.
4. RAG can reduce unsupported or generic answers, but it does not eliminate the need for human review.

## Connection to AI Native SDLC

RAG is highly relevant to AI Native SDLC because software teams rely on large amounts of project knowledge. RAG can help retrieve requirements, design decisions, code documentation, test cases, and historical defect information to support AI-assisted engineering workflows.
