# Sample Documents for RAG Experiment

## Document 1: AI Native SDLC Overview

AI Native SDLC refers to software development lifecycle activities that are designed with AI assistance as a native capability. This may include AI-assisted requirements engineering, design support, code generation, testing, documentation, deployment support, and maintenance.

## Document 2: Requirements Engineering

LLMs can assist with requirements clarification, ambiguity detection, user story generation, and acceptance criteria drafting. Human review is required to validate accuracy and domain fit.

## Document 3: Testing

AI tools can support test case generation, test data suggestions, regression test selection, and defect analysis. Generated test cases should be reviewed for correctness, completeness, and traceability.

## Document 4: Documentation

RAG can support documentation by retrieving relevant project knowledge and producing grounded drafts. This is useful for onboarding, user guides, API references, and release notes.
