# RAG Evaluation Checklist

Use this checklist to review RAG-based outputs.

## Retrieval Quality

- [ ] Are the retrieved chunks relevant to the query?
- [ ] Are important source documents missing?
- [ ] Are the chunks too short or too long?
- [ ] Is the source of each chunk identifiable?

## Answer Quality

- [ ] Does the answer use retrieved context?
- [ ] Does the answer avoid unsupported claims?
- [ ] Is the answer clear and concise?
- [ ] Are source references included where possible?

## SDLC Suitability

- [ ] Is the output useful for the intended SDLC activity?
- [ ] Does it preserve traceability to project artifacts?
- [ ] Does it require expert review before use?
- [ ] Are confidentiality and data governance concerns addressed?
