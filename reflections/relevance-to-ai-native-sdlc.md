# Relevance of Course Learning to AI Native SDLC

My PhD research area is AI Native Software Development Life Cycle. The course learning is relevant because Generative AI and Large Language Models can support multiple stages of the SDLC, including requirements analysis, design assistance, code generation, testing, documentation, maintenance, and knowledge retrieval.

## Key Connections

1. **Prompt Engineering and Requirements Engineering**  
   Prompt engineering is relevant to converting user needs into structured requirements, user stories, acceptance criteria, and design inputs.

2. **LLM API Integration and AI-Native Tooling**  
   API-based LLM integration supports the development of AI-native developer tools, copilots, coding assistants, documentation assistants, and review assistants.

3. **RAG and Knowledge-Driven Development**  
   Retrieval-Augmented Generation can support software teams by retrieving project documentation, codebase knowledge, design documents, and engineering guidelines.

4. **LLM Application Development and SDLC Automation**  
   LLM-powered applications can be used to automate or augment SDLC activities such as test case generation, code explanation, defect analysis, and release documentation.

## Summary

The hands-on learning completed in this course supports the practical foundation required to investigate AI Native SDLC as a PhD research area.
